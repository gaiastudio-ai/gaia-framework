/gaia-tech-debt-review

user-facing
Category:
Sprint Management
Lifecycle phase:
4 -- Implementation
Arguments:
None

What it does

/gaia-tech-debt-review scans all story files for technical debt findings, classifies them (design, code, test, infrastructure), assigns stable identifiers, scores by impact and risk, computes aging, and generates a rolling dashboard. The dashboard includes trend comparison against the previous review, overdue alerts, and recommended actions.

When to use it

  • You want to assess the current state of technical debt across the project.
  • Sprint planning needs data on which debt items are overdue or blocking velocity.
  • A retrospective flagged growing tech debt as a concern.

Prerequisites

  • Story files with findings. At least some stories in .gaia/artifacts/implementation-artifacts/ should have populated Findings tables. If none exist, the command reports "No technical debt detected."

How to invoke

/gaia-tech-debt-review

No arguments needed. Discovers all inputs automatically.

What it does step by step

  1. Scan debt sources Reads story file frontmatter and Findings sections. Validates triage targets. Identifies resolved items.
  2. Classify and deduplicate Merges duplicate findings, assigns stable TD-{N} identifiers, and categorizes each item as DESIGN, CODE, TEST, or INFRASTRUCTURE.
  3. Score and prioritize Rates each item on Impact, Effort, and Risk of Inaction. Computes a debt score and maps to priority: FIX NOW, PLAN NEXT, or TRACK.
  4. Calculate aging Computes age in sprints for each item. Auto-escalates aged items and flags overdue debt.
  5. Generate dashboard Writes the rolling dashboard to .gaia/artifacts/implementation-artifacts/tech-debt-dashboard.md with summary metrics, category table, aging histogram, top-10 table, and trend comparison.
  6. Recommend actions Suggests specific next-step commands based on the findings.

Inputs

InputSourceDescription
Story files.gaia/artifacts/implementation-artifacts/Frontmatter and Findings sections (token-budget-protected scan).
Sprint trackingsprint-status.yamlCurrent sprint ID for aging calculations.
Previous dashboardtech-debt-dashboard.mdPrior TD-{N} IDs and totals for trend comparison.

Outputs

OutputLocationDescription
Tech debt dashboard.gaia/artifacts/implementation-artifacts/tech-debt-dashboard.mdRolling dashboard with scores, aging, and trends.

Example session

> /gaia-tech-debt-review

Scanning story files... 23 stories scanned.
  12 tech-debt findings, 3 medium-severity bugs.
  Merged 4 duplicate findings into 2 items.
  2 STALE TARGET items verified: 1 resolved (excluded), 1 still active.

Classification:
  DESIGN: 3 items | CODE: 5 items | TEST: 2 items | INFRASTRUCTURE: 1 item

Top 3 by score:
  TD-7  Hardcoded config paths (CODE)       Score: 8  FIX NOW
  TD-3  Missing retry logic (DESIGN)        Score: 7  FIX NOW
  TD-12 No integration test for webhooks    Score: 5  PLAN NEXT

Aging: 2 items overdue (FIX NOW, open > 1 sprint).

Dashboard written to .gaia/artifacts/implementation-artifacts/tech-debt-dashboard.md.
Trend: +2 items vs previous review (11 items last review).

What to run next

  • /gaia-triage-findings -- reassign items with stale or missing targets.
  • /gaia-correct-course -- inject FIX NOW items into the current sprint.
  • /gaia-retro -- review the debt dashboard during the retrospective.

Troubleshooting

"No technical debt detected"

No story files have populated Findings tables. This is normal for new projects or after a thorough triage.
